🧱 All HTML5 Tags by Category

1. Structure

TagDescriptionExampleBest PracticeMDN
<html> Root element of an HTML document. <html lang="en">...</html> Always set the lang attribute for accessibility. MDN
<head> Container for metadata, links, and scripts. <head>...</head> Include meta tags for charset and viewport. MDN
<title> Document title (shown in browser tab). <title>My Page</title> Use a unique, descriptive title for SEO. MDN
<body> Main content of the document. <body>...</body> Only one <body> per page. MDN
<meta> Metadata about the document (charset, viewport, etc.). <meta charset="UTF-8"> Always specify charset and viewport for modern sites. MDN
<link> Links external resources (CSS, icons, etc.). <link rel="stylesheet" href="style.css"> Use rel and href attributes correctly. MDN
<style> Embeds CSS styles in the document. <style>body { color: red; }</style> Prefer external CSS for large projects. MDN
<base> Specifies base URL for relative links. <base href="https://example.com/"> Place inside <head>; only one per page. MDN

2. Document Sections

TagDescriptionExampleBest PracticeMDN
<header> Introductory content or navigation links for a section/page. <header>...</header> Use once per section or page for clarity. MDN
<nav> Navigation links for the site or section. <nav>...</nav> Use for primary navigation, not every group of links. MDN
<main> Main content unique to the page. <main>...</main> Only one <main> per page for accessibility. MDN
<section> Thematic grouping of content, typically with a heading. <section>...</section> Use headings inside for structure and accessibility. MDN
<article> Self-contained, reusable composition (e.g., blog post). <article>...</article> Use for independent, distributable content. MDN
<aside> Content tangentially related to the main content (e.g., sidebar). <aside>...</aside> Use for sidebars, pull quotes, or related links. MDN
<footer> Footer for a section or page (e.g., copyright, links). <footer>...</footer> Use at the end of sections or the page. MDN
<address> Contact information for the author/owner. <address>Email: me@example.com</address> Use only for contact info, not physical addresses in content. MDN

3. Text Content

TagDescriptionExampleBest PracticeMDN
<h1>–<h6> Headings, <h1> is most important, <h6> least. <h1>Title</h1> Use headings in order, don’t skip levels for accessibility. MDN
<p> Paragraph of text. <p>Hello world!</p> Use for blocks of text, not for spacing. MDN
<br> Line break. Line 1<br>Line 2 Use sparingly; prefer CSS for spacing. MDN
<hr> Thematic break (horizontal rule). Text above<hr>Text below Use to separate content, not for decoration. MDN
<pre> Preformatted text (preserves whitespace). <pre> indented text</pre> Great for code blocks or ASCII art. MDN
<blockquote> Block-level quotation. <blockquote>Quote</blockquote> Use cite attribute for source if available. MDN
<q> Inline quotation. <q>Short quote</q> Browsers add quotation marks automatically. MDN
<cite> Title of a work (book, article, etc.). <cite>Moby Dick</cite> Use for titles, not for people’s names. MDN
<code> Inline code snippet. <code>let x = 5;</code> Use for code, commands, or technical terms. MDN
<abbr> Abbreviation or acronym. <abbr title="HyperText Markup Language">HTML</abbr> Always use the title attribute for clarity. MDN
<mark> Highlights text. <mark>important</mark> Use to highlight search terms or relevance. MDN
<time> Machine-readable date/time. <time datetime="2025-07-01">July 1, 2025</time> Use datetime for events, articles, etc. MDN
