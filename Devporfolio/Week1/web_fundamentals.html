ğŸŒ What Is the Web?

The **World Wide Web** is a collection of documents and resources connected by hyperlinks and URLs, all accessed via the internet using protocols like HTTP or HTTPS.

  • The web uses **client-server architecture**.
  • You (the client) use a **browser** to send a request to a **server**.
  • The server sends back HTML, CSS, JS, images, etc.

ğŸŒ The web is built on a few core ideas: hyperlinks, resource identifiers (URLs), HTML rendering, and stateless communication.

ğŸ§­ DNS & URLs

DNS (Domain Name System)

DNS translates **domain names** like google.com into IP addresses like 142.250.183.46 so your browser can connect to the correct server.

URL (Uniform Resource Locator)

https://www.example.com:443/path/to/page?query=value#section1
  • https: Protocol
  • www.example.com: Domain
  • :443: Port (optional)
  • /path/to/page: Resource path
  • ?query=value: Query string
  • #section1: Fragment ID (anchor)

ğŸ§  What Are Browsers?

Browsers (like Chrome, Firefox, Safari) are software applications that:

  • Parse HTML, CSS, and JavaScript
  • Render visual interfaces
  • Handle caching, cookies, and session data
  • Provide developer tools for debugging

Rendering Engine Examples:

  • Chrome / Edge: Blink
  • Safari: WebKit
  • Firefox: Gecko

ğŸ§± Basic HTML5 Structure

HTML (HyperText Markup Language) defines the **structure** of web pages.

<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>My Page</title>
  </head>
  <body>
    <h1>Welcome!</h1>
  </body>
</html>

Why use <meta> tags?

  • charset="UTF-8" ensures all languages/symbols display correctly
  • viewport makes site responsive on mobile

ğŸ”– Semantic HTML

Semantic tags clearly define the **meaning** of content blocks.

Tag Purpose
<header> Defines the top (logo, nav)
<nav> Navigation links
<main> Main content of the document
<section> Logical grouping of content
<article> Independent, reusable content (e.g., blog post)
<aside> Sidebar, tips, related info
<footer> Bottom area with metadata
ğŸ§  Search engines and screen readers **prefer semantic tags** because they make content more understandable and accessible.
